## 🧠 Project Description

**Functional Treat** is an advanced, enterprise-grade command-line analytical platform developed to systematically compute, parse, and mutate single-dimensional (1D arrays) and multi-dimensional (2D grids) numerical datasets. The primary technical focus of this system is to exploit the complete paradigms of functional programming in Python, avoiding external mathematical modules to emphasize algorithmic processing.

The structural engine operates within an explicit, loop-driven menu architecture that accepts user array inputs, creates dynamic summaries, and processes deep mathematical transformations. It showcases precise implementations of standard algorithmic concepts such as variable scoping (`global` variables), dynamic stack calculations (Recursion), inline functions (Lambdas), and complex parameter unpacking (`*args` and `**kwargs`).

By combining algorithmic logic with advanced string documentation (`__doc__`), this utility program behaves like an interactive terminal workspace. It effectively highlights how an engineer handles native multidimensional structures, builds modular application pipelines, and filters complex runtime collections.

## ⚙️ Core Modules & Functions

### 🔹 Array Input Management (`1D & 2D Modules`)
Enables clean console input parsing using optimized map splits (`list(map(int, ...))`) to map raw string rows straight into clean integer computational layers.

### 🔹 Built-in Summary Analytics
Performs instant architectural evaluations on active datasets calculating global totals, length metrics, extreme points (`max()`, `min()`), and arithmetic means.

### 🔹 Advanced Parameter Unpacking (`*args` / `**kwargs`)
Demonstrates robust software patterns by handling variable-length arguments alongside key-value variable dictionaries, including automated docstring extraction.

### 🔹 Deep Call Stack Subsystem (`Recursion`)
Features an independent operational mathematical module computing structural Factorials and sequence tracking loops via memory stack self-iteration.

### 🔹 Inline Logic Transformations (`Lambda Filter`)
Applies custom logic filtering sequences instantaneously using dynamic lambda evaluations to screen out structural arrays based on dynamic conditions.
